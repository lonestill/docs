Плагин — это один `.js` файл, который вызывает `BYM.register`. Внутри ты получаешь объект `api` со всеми модулями и функцию `getSetting` для чтения настроек.

## BYM.register

```js
BYM.register(meta, factory);
```

| Аргумент | Тип | Описание |
|----------|-----|----------|
| `meta` | object | Метаданные и список настроек плагина |
| `factory` | function | Получает `(api, getSetting)`, возвращает объект с `start` и `stop` |

## Метаданные

```js
BYM.register({
  id: 'my-plugin',           // уникальный идентификатор
  name: 'Мой плагин',        // имя в списке BYM
  version: '1.0.0',          // semver-строка
  description: 'Что делает', // короткое описание
  author: 'Имя',             // автор
  settings: [],              // массив настроек, см. /plugin-settings
}, (api, getSetting) => ({
  start() {},
  stop() {},
}));
```

| Поле | Обязательное | Описание |
|------|--------------|----------|
| `id` | да | Уникальный идентификатор. Используется для Storage и системных уведомлений |
| `name` | да | Имя, которое видит пользователь |
| `version` | да | Версия плагина |
| `description` | нет | Короткое описание |
| `author` | нет | Автор плагина |
| `settings` | нет | Массив настроек, см. [Настройки](/plugin-settings) |

## start и stop

Фабрика возвращает объект с двумя методами:

```js
return {
  start() {
    // вызывается при включении плагина
  },
  stop() {
    // вызывается при выключении и при перезагрузке файла
  },
};
```

`start` запускается, когда пользователь включает плагин или при старте YM, если плагин уже был включён. `stop` запускается при выключении.

<Note>
Многое чистится автоматически: CSS через `api.DOM.injectCSS`, патчи через `api.Patcher`, перехваты через `api.Net`, observers через `api.DOM.observe`. Подписки на `api.Events` и `api.Hotkeys` снимай вручную в `stop`.
</Note>

## Правила написания

- Один плагин — один файл. Импортов и require нет
- Не трогай DOM глобально без очистки в `stop`
- Сохраняй пользовательские данные через `api.Storage`, не через `localStorage`
- Для перехвата fetch используй `api.Net`, не патчи `window.fetch` напрямую
- Логируй через `api.Logger` — префикс имени плагина проставится автоматически

## Полный пример

```js
BYM.register({
  id: 'hello',
  name: 'Hello',
  version: '1.0.0',
  description: 'Тост при запуске YM',
}, (api) => {
  return {
    start() {
      api.UI.toast('Привет!');
      api.Logger.log('plugin started');
    },
    stop() {
      api.Logger.log('plugin stopped');
    },
  };
});
```
